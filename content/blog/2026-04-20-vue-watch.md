## 1. watch 监听基本类型的 ref

当 watch 用于监听 `ref` 定义的基本类型数据时，返回的新旧值分别为基本类型的复制值。

### 示例代码
```vue
<template>
  <div>
    <p>{{ sum }}</p>
    <button @click="changeSum">点我+1</button>
  </div>
</template>

<script setup>
import { ref, watch } from 'vue'

const sum = ref(0)
const stopWatch = watch(sum, (newValue, oldValue) => {
  console.log('sum 变了', oldValue, 'to', newValue)
  if (newValue >= 10) {
    stopWatch()
  }
})

function changeSum() {
  sum.value += 1
}
</script>
```

### 关键点
- `watch` 第一个参数是被监听的响应式来源。
- 回调参数 `newValue` 和 `oldValue` 代表变化后的值与变化前的值。
- 通过 `stopWatch()` 可以手动停止监听。

## 2. watch 监听对象类型的 ref

如果监听对象类型的数据，通常需要将 `deep: true` 作为第三个参数，否则对象内部属性的变化不会被侦测。

### 示例代码
```vue
<script setup>
import { ref, watch } from 'vue'

const person = ref({
  name: 'Alice',
  age: 30,
})

watch(
  person,
  (newValue, oldValue) => {
    console.log('person 变了', oldValue, 'to', newValue)
  },
  { deep: true },
)
</script>
```

### 关键点
- `person` 是 `ref` 包裹的对象类型。
- `{ deep: true }` 确保对象内部属性变化也会触发 watch。

## 3. immediate 选项

`immediate: true` 会让 watch 在注册时立即执行一次回调，并将当前值作为 `newValue` 传入，`oldValue` 为 `undefined`。

### 示例代码
```ts
watch(
  person,
  (newValue, oldValue) => {
    console.log('person 变了', oldValue, 'to', newValue)
  },
  { deep: true, immediate: true },
)
```

### 关键点
- `immediate: true` 会让 watch 在注册时立即执行一次回调。
- 第一次回调中，`newValue` 是当前的响应式值，`oldValue` 通常是 `undefined`。
- 这并不是“额外创建一次事件”，而是让你可以直接在注册时使用当前值。
- 该选项不改变后续正常的 watch 行为。

> `immediate: true` 的意思是：
>
> - 当你写 `watch(..., { immediate: true })` 时，Vue 会先执行一次回调。
> - 这次执行的目的是让你可以立刻根据当前值做一次处理，比如初始化数据、判断条件或同步状态。
> - 因为这时候还没有“旧值”，所以 `oldValue` 通常是 `undefined`。

## 4. 对象属性变化时 oldValue/newValue 的特性

当监听的是对象内部属性变化时，`oldValue` 和 `newValue` 可能会是同一个对象引用，因为对象本身没有被替换，只是内部属性发生了改变。

### 示例说明
- `person.value.age += 1` 后触发 watch：`oldValue` 与 `newValue` 依然引用同一个对象。
- `person.value = { name: 'Jack', age: 100 }` 后触发 watch：`oldValue` 与 `newValue` 为不同对象引用。

这就导致在深度监听对象时，要注意旧值并不总是可用于比较对象内部的变更历史。

## 5. 实践示例：对象属性变化与对象替换

```vue
<template>
  <div>
    <div>name: {{ person.name }}</div>
    <div>age: {{ person.age }}</div>
    <button @click="changeName">changeName</button>
    <button @click="changeAge">changeAge</button>
    <button @click="changePerson">changePerson</button>
  </div>
</template>

<script setup>
import { ref, watch } from 'vue'

const person = ref({
  name: 'Alice',
  age: 30,
})

function changeName() {
  person.value.name = 'Bob'
}

function changeAge() {
  person.value.age += 1
}

function changePerson() {
  person.value = {
    name: 'Jack',
    age: 100,
  }
}

watch(
  person,
  (newValue, oldValue) => {
    console.log('person 变了', oldValue, 'to', newValue)
  },
  { deep: true, immediate: true },
)
</script>
```

## 6. watch 的实际应用建议

- 监听基本类型时，使用普通 `ref` 即可。
- 监听对象类型时，若需要捕获内部属性变化，记得加上 `deep: true`。
- 如果只关心对象整体被替换，可以不加 `deep: true`。
- 如果需要在某个条件下停止监听，可以将 `watch` 返回值赋给一个变量，并在条件成立时调用它。
- 如果希望组件加载时马上执行一次回调，用 `immediate: true`。

## 7. 总结

1. `watch` 是 Vue 响应式系统里用于监听值变化的 API。
2. 监听 `ref` 基本类型与对象类型时，`newValue`/`oldValue` 的语义不同。
3. 对对象内部变化进行监听时，`deep: true` 是关键。
4. `immediate: true` 可以让 watch 在注册时先执行一次回调，这样就能直接使用当前值。
5. 调用 `stopWatch()` 可以手动停止监听，避免不必要的监听。
